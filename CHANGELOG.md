# Changelog

## 0.5.5

- 识别 Web 端「静默限流」：点击关注后按钮不变，连续 2 次则自动暂停
- 扩展限流/封控文案检测（今天关注过多、操作频繁等）
- 文档补充：Web 常无提示，手机端才会提示「今天关注过多」

## 0.5.4

- 精确模式强制展示昵称字符全等，禁止部分匹配
- 精确模式禁止仅按抖音号匹配
- 禁止横向绑定邻卡「关注」按钮
- 点击前再次校验展示名

## 0.5.3

- 空昵称自动跳过，不再中断流水线
- 加强完全精确匹配与点击前复核

## 0.5.2

- 成功关注 / 已关注后随机冷却 10–15 秒再跳下一个，降低验证码风险

## 0.5.1

- 优先匹配展示昵称，抖音号同名降权

## 0.5.0

- 主通道改为 DOM 精确文本搜索目标昵称（不限第一名）
- 双重核验关注结果
- 取消 5 分钟硬锁，改为短冷却

## 0.4.x

- 整文件重写：单标签搜索、关注按钮锚点、手动/自动模式、问题项重跑

## 0.1–0.3

- 初版半自动方案与迭代（抓取、搜索、匹配、限速）
